What is Lua?

Lua is a fast, lightweight, and embeddable scripting language designed for flexibility and ease of integration. Originally developed in 1993 at the Pontifical Catholic University of Rio de Janeiro, Lua has since become a popular choice for game development, embedded systems, and automation.

Technical Overview

Scoping

Lua uses lexical scoping, meaning variable visibility is determined at the time of writing the code. It supports:

  • Global Scope: Variables without local are accessible throughout the program.
  • Local Scope: Variables declared with local are confined to their block or function.
  • Upvalues: Functions can capture variables from their enclosing scope, enabling closures.

x = 10  -- Global scope

function example()
    local y = 5  -- Local scope, only accessible inside `example`
    print(x)  -- Can access global `x`
    print(y)  -- Can access local `y`
end

example()
print(y)  -- ERROR! `y` is not accessible outside the function

Garbage Collection

Lua uses an automatic mark-and-sweep garbage collector:

  • Mark Phase: Identifies reachable objects.
  • Sweep Phase: Removes unmarked objects from memory.
  • Generational Collection: Optimizes performance by categorizing objects based on lifespan.

Compilation & Interpretation

Lua is an interpreted language, but it compiles scripts into bytecode before execution:

  • Compilation to Bytecode: Lua source code is converted into bytecode using the built-in compiler.
  • Execution by Virtual Machine: The Lua interpreter runs the bytecode on the Lua Virtual Machine (Lua VM).
  • Just-in-Time Compilation: LuaJIT provides JIT compilation for significant performance improvements.
